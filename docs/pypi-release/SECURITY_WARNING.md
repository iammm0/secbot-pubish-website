## 授权测试

在使用这些工具之前，请确保：

1. ✅ 已获得目标系统所有者的明确书面授权
2. ✅ 已签署合法的渗透测试协议
3. ✅ 已了解并遵守相关法律法规
4. ✅ 已制定详细的测试计划和范围
5. ✅ 已建立适当的日志和报告机制

## 使用建议

1. **仅在测试环境中使用**：除非已获得明确授权，否则仅在您拥有或已获得授权的测试环境中使用
2. **记录所有操作**：所有攻击测试操作都会被记录到数据库中，请妥善保管
3. **遵守测试范围**：严格按照授权范围进行测试，不要超出授权范围
4. **及时报告**：发现漏洞后，及时向授权方报告，不要恶意利用
5. **保护数据**：测试过程中获取的任何敏感数据都应妥善保管

## 功能说明

### 端口扫描
- 用于发现目标系统开放的端口和服务
- 可以帮助识别潜在的攻击面

### 漏洞扫描
- 检测常见的Web漏洞（SQL注入、XSS、目录遍历等）
- 识别服务版本和已知漏洞

### 攻击测试
- **暴力破解**：测试登录系统的密码强度
- **DoS测试**：测试系统的抗压能力
- **SQL注入**：测试Web应用的SQL注入防护
- **XSS测试**：测试跨站脚本攻击防护

### 定时任务
- 支持定时执行攻击测试
- 用于持续的安全监控
